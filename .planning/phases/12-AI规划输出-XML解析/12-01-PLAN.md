<objective>
Rewrite parseChaptersXml() from Jackson XML to DOM parsing, adding <ch> character tag extraction, NameMatchUtil-based character ID resolution, and plannedCharacters persistence.

Purpose: The current Jackson XML parser (ChapterPlanXmlDto) cannot handle nested same-name <ch> tags inside <o> elements. DOM parsing follows the proven WorldviewXmlParser pattern and extracts both chapter fields and character planning data in a single pass. This is the core data pipeline change for Phase 12.

Output: Working DOM parser with character tag extraction, name matching, and persistence; comprehensive unit tests.
</objective>
