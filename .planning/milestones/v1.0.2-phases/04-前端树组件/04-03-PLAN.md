<objective>
Fix the recursive component CRUD bug in FactionTree.vue so that add and edit operations work correctly at all tree depths.

Purpose: The current self-referencing `<FactionTree>` component creates separate component instances at each depth, each with its own isolated state. When a user clicks add or edit on a node at depth >= 1, the child instance emits events without context data, so the root instance cannot process the request correctly. This gap closure makes all CRUD operations functional at every depth.

Output: Modified FactionTree.vue with event-bubbling approach -- child instances pass full context (parentId for add, full Faction node for edit) through the event chain to the root instance, and add/edit forms are visible in child instances.
</objective>
