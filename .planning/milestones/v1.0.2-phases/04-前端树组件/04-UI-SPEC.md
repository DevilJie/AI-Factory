## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, badge padding (px-1 py-0.5) |
| sm | 8px | Compact element spacing, node vertical padding (py-2) |
| md | 16px | Default element spacing, section padding (p-6), input padding (px-4 py-3) |
| lg | 24px | Section header margins (mb-4, mb-6), button padding (px-6 py-3) |
| xl | 32px | Layout gaps between cards (gap-6 = 24px, gap-8 = 32px) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Tree indent per level: 20px per depth (matching GeographyTree.vue `paddingLeft = 20 + deep * 20`)
- Icon-only action button touch target: p-1 (4px padding) within a 3.5x3.5 icon, yielding ~14px -- acceptable for secondary hover-revealed actions only
- Node row spacing: space-y-0.5 (2px gap between rows), matching GeographyTree

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / node name | 14px (text-sm) | 500 (font-medium) for top-level names, 400 for children | default (1.5) |
| Label / badge text | 12px (text-xs) | 400 (normal) | default |
| Section heading | 14px (text-sm) | 500 (font-medium) | default |
| Description text | 12px (text-xs) | 400 (normal) | default |

Notes:
- Maximum 2 weights: 400 (normal) and 500 (medium), matching GeographyTree.vue usage
- Maximum 2 sizes: 14px (text-sm) for names and headings, 12px (text-xs) for descriptions, badges, labels
- Top-level faction names use font-medium; child faction names use default weight (visual hierarchy via weight, not size)

## Interaction Patterns

### Tree Navigation
- Expand/collapse: click chevron icon (ChevronRight collapsed, ChevronDown expanded)
- Child nodes indented 20px per depth level
- Only nodes with children show chevron; leaf nodes show blank spacer (w-5)

### Node Display (Top-Level, deep === 0)
```
[正派] 紫阳宗 · 仙道                          [+ ✏ 🗑]
  测试描述内容
```
- Type badge: colored rounded pill (px-1.5 py-0.5 text-xs rounded)
- Name: text-sm font-medium
- Power system: text-xs text-gray-400, dot separator " · "
- Description: text-xs text-gray-400 mt-0.5 (if present)

### Node Display (Child, deep > 0)
```
  内门                                          [+ ✏ 🗑]
    内门描述
```
- No type badge, no power system label (inherited from root)
- Name: text-sm default weight (not font-medium)
- Description: text-xs text-gray-400 (if present)

### Inline Editing (Name + Description)
1. Click edit button (Edit3 icon) on any node
2. Name becomes input field (text-sm), description becomes textarea (text-xs, rows=2)
3. Save/Cancel buttons appear (px-2 py-1 text-xs)
4. Enter key saves, Escape key cancels
5. Type and power system selectors expand below the node for top-level factions only

### Type Selector (Top-Level Only)
1. In edit mode for top-level node, a button group appears below the name input
2. Three buttons: 正派 (green), 反派 (red), 中立 (gray)
3. Selected button highlights with type color; unselected show bg-gray-100
4. Confirm (确定) / Cancel (取消) buttons

### Power System Selector (Top-Level Only)
1. In edit mode for top-level node, a dropdown/selector appears
2. Lists all power systems from getPowerSystemList()
3. Includes "无" option to clear the field
4. Resolves ID to display name using local Map<number, string>

### Add Node
1. Click Plus icon on a node (or root-level "添加势力" button)
2. Add form appears below the node (or at top for root)
3. Form contains: name input (required) + description textarea (optional)
4. For root add: form includes name + description only (type/powerSystem set after creation via edit)
5. For child add: form includes name + description only (type/powerSystem inherited)
6. Enter key submits, Escape key cancels

### Delete Node
1. Click Trash2 icon (hover-revealed)
2. Browser confirm() dialog with appropriate message (leaf vs parent)
3. On confirm: call delete API, refresh tree, show success toast
4. On cancel: no action

### Action Buttons Visibility
- Add, Edit, Delete buttons hidden by default (opacity-0)
- Reveal on node row hover (group-hover:opacity-100)
- Transition: transition-opacity (smooth fade)

## States

### Loading State
- Centered Loader2 icon (w-6 h-6 text-blue-500 animate-spin) with py-8 vertical padding
- Displayed during initial data fetch

### Empty State
- Centered text (text-center py-8)
- Heading: "暂无势力阵营" (text-gray-400 dark:text-gray-500)
- Body: "点击上方「添加势力」按钮开始构建势力体系"

### Error State
- Toast notification via error() from @/utils/toast
- No inline error display; tree remains in last known good state

### Disabled State (during AI generation)
- All action buttons receive disabled:opacity-50
- Add root button receives :disabled="disabled" prop
- Input fields not interactive
- Display only mode

### Data Refresh
- After AI generation: factionTreeRef.value?.refresh() called in handleGenerate() and restoreGeneratingState()
- After any CRUD operation: full tree reload via loadData()
- Power system list loaded in parallel with faction tree via Promise.all()
