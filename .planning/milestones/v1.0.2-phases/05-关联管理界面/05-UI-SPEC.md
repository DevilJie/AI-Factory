## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, badge padding (px-1 py-0.5) |
| sm | 8px | Compact element spacing, node vertical padding (py-2) |
| md | 16px | Default element spacing, drawer content padding (p-6), input padding (px-3 py-2) |
| lg | 24px | Section margins between form and list (mb-4), drawer header padding (px-6 py-4) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Drawer max width: max-w-2xl (672px) -- matching SettingsDrawer.vue
- Tab button padding: px-4 py-2 -- matching SettingsDrawer tab buttons
- List item vertical spacing: space-y-2 (8px) between association rows
- Tree indent per level in region selector: 16px per depth (region.deep * 16)
- Icon-only action button touch target: p-1 (4px padding) -- same as FactionTree node actions

Source: Phase 04 UI-SPEC spacing scale, extended for Drawer-specific tokens.

## Interaction Patterns

### Opening the Drawer (D-01)
1. Each faction node in FactionTree shows a Link2 icon button in the hover-revealed action button area
2. Clicking the Link2 button emits an event (e.g., `openDrawer` with the node data) up to the root FactionTree instance
3. Root FactionTree sets `drawerOpen = true` and `drawerFaction = node`, rendering FactionDrawer
4. Drawer slides in from the right with translate-x-full -> translate-x-0 transition (300ms ease-out)
5. Overlay fades in with opacity transition (300ms)
6. Body scroll is locked (overflow: hidden) while drawer is open

### Closing the Drawer
1. Click X button in drawer header
2. Click overlay (bg-black/50)
3. Press Escape key
4. Drawer slides out, overlay fades out, body scroll restored
5. Drawer state (activeTab, form inputs) is reset on close via v-if destruction of tab content

### Tab Navigation (D-02)
1. Three tabs in the drawer header: "势力关系", "人物关联", "地区关联"
2. Tab bar uses the same bg-gray-100 rounded-xl p-1 pattern as SettingsDrawer
3. Active tab: bg-white shadow-sm with icon
4. Switching tabs destroys previous tab content (v-if) and creates new tab content
5. Each tab independently loads its data when created (watch on factionId or onMounted)

### Data Loading (D-03)
1. When Drawer opens, the active tab loads its association list via GET API
2. Concurrent data loads where possible (e.g., faction list + relations for relation tab)
3. Loading state: centered Loader2 spinner (w-5 h-5 text-blue-500 animate-spin)
4. After any CRUD operation: reload the current tab's list, show success toast

### Relation Tab Pattern (D-04, D-05, D-06, D-07)

**Form area (fixed at top):**
1. Target faction dropdown: `<select>` populated from flat faction list (all factions, excluding current)
2. Relation type selector: three buttons (盟友/敌对/中立) with type badge colors
3. Description textarea: optional, 1 row
4. Submit button: "添加"

**Bidirectional logic (D-05):**
1. On add: call POST twice -- once with factionId=A/targetFactionId=B, once with factionId=B/targetFactionId=A
2. If the second call fails, show error toast but the first relation persists
3. On delete: find the reverse relation (GET relations for targetFactionId, match by targetFactionId + relationType), delete both records

**List area (below form):**
1. Each row shows: target faction name + relation type badge + description (truncated) + delete button
2. Rows stacked vertically with space-y-2
3. Delete button: Trash2 icon, hover reveals, confirm dialog before delete

### Character Tab Pattern (D-08, D-09, D-10)

**Form area (fixed at top):**
1. Character search input: text input with local keyword filtering (no debounce needed, data loaded once)
2. Filtered dropdown appears below input when typing, shows character name + role
3. Already-associated characters shown with "(已关联)" suffix but still selectable (D-10 allows multiple associations)
4. Role field: `<input list="role-options">` with `<datalist>` containing preset values
5. Preset roles: 掌门, 副掌门, 长老, 弟子, 护法, 执事
6. Submit button: "关联人物"

**List area (below form):**
1. Each row shows: character name + role badge + delete button
2. Role displayed as colored text label (text-sm)
3. Delete: confirm dialog "移除「{name}」的关联？"

### Region Tab Pattern (D-11, D-12)

**Form area (fixed at top):**
1. Expandable tree selector rendered inline (not a dropdown -- stays open)
2. Tree shows all regions with indentation (deep * 16px padding-left)
3. Each node has a checkbox; checked nodes are selected for batch add
4. Chevron toggle for expand/collapse of child nodes
5. Already-associated regions shown as checked + disabled (cannot re-add)
6. Submit button: "添加地区" -- batch creates all checked new regions

**List area (below tree selector):**
1. Each row shows: region name (with hierarchy path, e.g., "大陆 > 王国 > 城市") + delete button
2. Delete: confirm dialog "移除地区「{name}」的关联？"

### Tree Selector Behavior
1. Default: first-level nodes expanded, deeper nodes collapsed
2. Click chevron to toggle expand/collapse
3. Checkbox per node (no "select all")
4. Already-associated regions: checkbox checked and disabled, grayed out

## States

### Loading State (per tab)
- Centered Loader2 spinner (w-5 h-5 text-blue-500 animate-spin py-8)
- Displayed during data fetch when tab first mounts or after CRUD operation

### Empty State (per tab)
- Relations: "暂无势力关系，选择目标势力后添加" (text-center py-8 text-gray-400)
- Characters: "暂无关联人物，搜索并关联角色"
- Regions: "暂无关联地区，选择地区后添加"
- Centered text, no icon needed

### Error State
- Toast notification via error() from @/utils/toast
- Form remains populated for retry; list shows last known good state

### Form Validation
- Relation: target faction required, relation type required
- Character: character selection required, role optional (empty string allowed)
- Region: at least one checkbox required for batch add
- Invalid submit: error toast "请选择目标势力" / "请选择人物" / "请至少选择一个地区"

### Disabled State
- During AI generation: FactionTree passes disabled prop, Link2 button receives disabled:opacity-50
- During Drawer CRUD operations: submit button shows loading state (Loader2 inline)

### Search Empty State (Character Tab)
- When keyword filter yields no results: "未找到匹配的人物" (text-center py-4 text-gray-400)
- When character list itself is empty: "暂无人物数据，请先创建角色"

### Drawer Transition State
- Opening: overlay opacity 0->1, panel translate-x-full->translate-x-0 (300ms ease-out)
- Closing: reverse transitions (300ms ease-in)
- Tab content: v-if keyed by activeTab -- instant swap, no animation between tabs
