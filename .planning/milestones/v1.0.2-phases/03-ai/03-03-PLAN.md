<objective>
Migrate all 8 files (10 call sites) that call worldview.getForces() to first call factionService.fillForces(worldview), ensuring the transient forces field is populated from the structured faction tables.

Purpose: Without fillForces(), the forces field is always null (marked @TableField(exist=false)), so all prompt construction would produce no faction information. This migration bridges the gap between structured storage and prompt text.

Output: 8 modified files, each with FactionService injected and fillForces() called before getForces().

Note on scope: D-09 lists 6 call sites (PromptTemplateBuilder, ChapterPromptBuilder, PromptContextBuilder, ChapterFixTaskStrategy, VolumeOptimizeTaskStrategy, OutlineTaskStrategy). This plan also covers 2 additional files (ChapterGenerationTaskStrategy, ChapterService) discovered during research that also call getForces(). These are included because they would otherwise receive null forces at runtime, producing incomplete prompts or chapter content.
</objective>
