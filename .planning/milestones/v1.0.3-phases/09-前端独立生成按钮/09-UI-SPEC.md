## Component Inventory

### New: AI Generate Button (inline, not a separate component)

Each of the 3 child components (GeographyTree, PowerSystemSection, FactionTree) adds an inline button in its header area. The button pattern is identical across all three.

**Button markup specification:**

```html
<button
  @click="$emit('generate')"
  :disabled="disabled"
  class="flex items-center gap-1 px-3 py-1.5 text-xs font-medium text-white bg-gradient-to-r from-green-500 to-teal-500 rounded-lg hover:from-green-600 hover:to-teal-600 transition-colors disabled:opacity-50"
>
  <Loader2 v-if="generatingSelf" class="w-3 h-3 animate-spin" />
  <Sparkles v-else class="w-3 h-3" />
  {{ generatingSelf ? '生成中...' : 'AI生成' }}
</button>
```

**Props each child component must accept:**

| Prop | Type | Purpose |
|------|------|---------|
| `disabled` | `boolean` | Existing -- disables all interaction when any generation is running |
| `generatingSelf` | `boolean` | New -- true when THIS specific module is generating (controls spinner + text) |

**Events each child component must emit:**

| Event | Payload | Purpose |
|-------|---------|---------|
| `generate` | none | Parent (WorldSetting.vue) handles API call + polling + refresh |

### Header Layout (existing, modified)

The header `div` currently uses `flex items-center justify-between`. The right side currently has a single blue "Add" button. The new AI button is placed BEFORE the "Add" button, creating a button group of two.

```html
<div class="flex items-center gap-2">
  <!-- NEW: AI Generate Button -->
  <button @click="$emit('generate')" ...>AI生成</button>
  <!-- EXISTING: Add Button -->
  <button @click="showAddForm(null)" ...>添加区域</button>
</div>
```

### Top-Level AI Button (existing, modified)

The existing "AI生成" button in WorldSetting.vue header gains an additional disabled condition:

```html
<button
  @click="handleGenerate"
  :disabled="generating || !!generatingModule"
  class="flex items-center gap-2 px-4 py-2 text-sm font-medium text-white bg-gradient-to-r from-green-500 to-teal-500 rounded-lg hover:from-green-600 hover:to-teal-600 transition-colors disabled:opacity-50"
>
  <Loader2 v-if="generating" class="w-4 h-4 animate-spin" />
  <Sparkles v-else class="w-4 h-4" />
  AI生成
</button>
```

The `:disabled` condition changes from `generating` to `generating || !!generatingModule` to enforce D-06.

## Interaction States

### State Matrix

| State | AI Button (active module) | AI Button (sibling module) | AI Button (top-level) | Module tree/list | Add Button |
|-------|--------------------------|---------------------------|----------------------|------------------|------------|
| Idle | Enabled, "AI生成" | Enabled, "AI生成" | Enabled, "AI生成" | Interactive | Enabled |
| Module generating | Disabled, spinner, "生成中..." | Disabled, opacity-50 | Disabled | Disabled (via `disabled` prop) | Disabled |
| Whole generating | N/A (all hidden behind overlay) | N/A | Disabled, spinner | Hidden behind overlay | Hidden |
| API error (dependency) | Reverts to idle | Unaffected | Reverts to idle | Reverts to interactive | Reverts to enabled |

### Mutual Exclusion (D-05, D-06)

A single `generatingModule` ref in WorldSetting.vue tracks which module is generating:

```
generatingModule: ref<'geography' | 'powerSystem' | 'faction' | null>
```

- When `generatingModule` is set to any non-null value: all 3 module AI buttons disabled, top-level AI button disabled
- When `generatingModule` is null: all buttons enabled (unless `generating` is true for whole-worldview)
- When `generating` is true (whole-worldview): full-page overlay covers everything

### Task Recovery (D-07, D-08)

localStorage keys per module:

| Module | Key Format |
|--------|-----------|
| Geography | `ai-factory-geography-generate-{projectId}` |
| Power System | `ai-factory-powerSystem-generate-{projectId}` |
| Faction | `ai-factory-faction-generate-{projectId}` |

On page load, each module's localStorage key is checked independently. If a task is found and not timed out (10 minutes), resume polling for that module only. Other modules remain fully interactive.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (idle) | `AI生成` (with Sparkles icon, green gradient button) |
| Primary CTA (loading) | `生成中...` (with Loader2 spinner, disabled green gradient button) |
| Geography success toast | `地理环境生成成功` |
| Power System success toast | `力量体系生成成功` |
| Faction success toast | `势力阵营生成成功` |
| Module timeout error | `AI生成{模块名}超时` |
| Module poll failure | `{errorMessage from task}` or `AI生成{模块名}失败` |
| Dependency error (geography missing) | Backend returns: `请先生成地理环境数据` -- Axios interceptor auto-toasts |
| Dependency error (power system missing) | Backend returns: `请先生成力量体系数据` -- Axios interceptor auto-toasts |
| Top-level AI button | `AI生成` (unchanged per D-03) |

**Error handling rule:** Do NOT call `error()` in the catch block for API-level errors (HTTP response with `code !== 0`). The Axios interceptor in `request.ts` already shows the toast. Only call `error()` for poll-level failures (task failed/cancelled/timeout). This prevents double-toast.
