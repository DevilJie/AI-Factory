3. Order 3, name "保存力量体系", type "save_power_system"

- Get all power systems for project: `List<NovelPowerSystem> systems = powerSystemService.listByProjectId(projectId)`

- Log "清理旧力量体系数据完成, projectId={}, 删除数量={}"
- Return StepResult.success with progress 100

**generate_power_system step:**
- Same pattern as GeographyTaskStrategy: load project, extract config, build variables
- Template code: `"llm_power_system_create"`
- Build AIGenerateRequest with requestType "llm_power_system_create"
- Store aiResponse in sharedData
- Return StepResult.success

**save_power_system step:**
- Get aiResponse from sharedData; null check
- Parse using xmlParser: `WorldSettingXmlDto worldSetting = xmlParser.parse(aiResponse, WorldSettingXmlDto.class)` (same as WorldviewTaskStrategy line 335)
- If worldSetting.getSystems() is null or empty, log warning and return success (no power systems to save)
- Copy the save logic from WorldviewTaskStrategy.savePowerSystems (lines 491-556) BUT with a critical difference:
  - **Do NOT create NovelWorldviewPowerSystem associations.** Per research Pitfall 2, the worldview entity may not exist during independent generation. Just save the power system data (system + levels + steps) without the worldview association. Phase 8 will handle associations.
- The save logic:
  1. For each CultivationSystem in worldSetting.getSystems().getSystemList():
     - Insert NovelPowerSystem (projectId, name, sourceFrom, coreResource, cultivationMethod, description)
     - For each CultivationLevel: insert NovelPowerSystemLevel (powerSystemId, level, levelName, description, breakthroughCondition, lifespan, powerRange, landmarkAbility)
     - For each step in level: insert NovelPowerSystemLevelStep (powerSystemLevelId, level, levelName)
  2. Do NOT insert NovelWorldviewPowerSystem records
- Log "力量体系入库完成, projectId={}, 共{}套体系"
- Return StepResult.success

**Important notes:**
- Do NOT create worldview-power-system association records (NO `worldviewPowerSystemMapper` usage)
- The `WorldSettingXmlDto.Systems` and inner classes (`CultivationSystem`, `CultivationLevel`, etc.) are already defined and parsed by XmlParser
- Use LocalDateTime.now() for timestamps on insert
