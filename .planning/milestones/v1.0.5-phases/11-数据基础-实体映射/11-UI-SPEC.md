## Scope Declaration

**Phase 11 has no frontend deliverables.** All work is backend-only:

1. Database migration (ALTER TABLE for `planned_characters` column)
2. Entity field mapping (`NovelChapterPlan.plannedCharacters`, `NovelChapterPlan.characterArcs`)
3. DTO merge (delete unused `common.xml.ChapterPlanXmlDto`)
4. Utility class (`NameMatchUtil` -- three-tier Chinese name matching)

No new Vue components, no new views, no CSS changes, no UI interactions.

**Rationale:** Phase 11 scope is explicitly defined in CONTEXT.md and ROADMAP.md as data-layer infrastructure. The "UI hint: yes" flag in the roadmap applies only to Phase 14 (前端展示 + 闭环验证). Phase 11 success criteria are verifiable via API responses and unit tests alone.

## Downstream Reference

Phase 11 establishes data structures that Phase 14 (前端展示 + 闭环验证) will consume. When Phase 14's UI-SPEC is created, it should reference:

1. `planned_characters` JSON schema (5 fields: characterName, roleType, roleDescription, importance, characterId)
2. `ChapterPlanDto.plannedCharacters` field -- the API response field for frontend consumption
3. `NameMatchUtil` matching result -- `characterId` null means unmatched, `characterName` preserved
4. `importance` field values (high/medium/low) -- recommended for visual weight differentiation in Phase 14 cards
5. `roleType` field values (protagonist/supporting/antagonist/npc) -- recommended for color coding in Phase 14
